# Operating Blueprint — Come Funziona l'Azienda

**Cartella:** `knowledge-core/operating-blueprint/`  
**Scopo:** Il manuale operativo completo dell'azienda

---

## A cosa serve questa cartella

Se il Manifesto risponde a "perché esistiamo", l'Operating Blueprint risponde a "come funzioniamo".

Questo è il **manuale dell'azienda** — il documento che un nuovo collaboratore (umano o AI) deve leggere per capire come gira il business. Contiene le regole operative, i sistemi in uso, le responsabilità, i flussi di lavoro standard.

L'obiettivo è che chiunque — anche qualcuno che non ha mai lavorato con noi — possa leggere l'Operating Blueprint e iniziare a operare correttamente entro poche ore.

## La differenza con i Processi

L'Operating Blueprint descrive **il sistema nel suo insieme** — come le parti si collegano.  
La cartella `processi/` contiene le **istruzioni passo-passo** per ogni attività specifica.

Pensa all'Operating Blueprint come alla mappa dell'azienda. I Processi sono le strade dettagliate.
