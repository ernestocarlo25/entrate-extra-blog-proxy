# Processi — Le Istruzioni Operative

**Cartella:** `knowledge-core/processi/`  
**Scopo:** Istruzioni passo-passo per ogni attività ricorrente dell'azienda

---

## A cosa serve questa cartella

Un processo documentato è un'attività che chiunque può eseguire — anche qualcuno che non ha mai partecipato a quella specifica attività prima.

I processi eliminano la dipendenza dalla memoria delle singole persone e garantiscono che ogni attività venga eseguita allo stesso modo, ogni volta.

---

## Come sono strutturati i processi

Ogni processo è un file separato. Ogni file segue questa struttura:

```markdown
# [Nome Processo]

**Frequenza:** settimanale / mensile / ad hoc  
**Responsabile:** CEO / AI  
**Tempo stimato:** X minuti  
**Prerequisiti:** cosa serve prima di iniziare

## Passaggi

1. Primo passaggio
2. Secondo passaggio
...

## Checklist di completamento

- [ ] Verifica 1
- [ ] Verifica 2

## Note e casi particolari
```

## La differenza con l'Operating Blueprint

L'Operating Blueprint spiega **come funziona il sistema nel suo insieme**.  
I Processi spiegano **come eseguire una specifica attività passo dopo passo**.

Il Blueprint è la mappa. I Processi sono le istruzioni per navigare ogni strada.
