# Session Report — Il Diario delle Sessioni di Lavoro

**Cartella:** `knowledge-core/session-report/`  
**Scopo:** Documentare ogni sessione di lavoro significativa con il sistema AI

---

## A cosa serve questa cartella

Ogni sessione di lavoro con l'AI produce decisioni, codice, contenuti, scoperte. Se non viene documentata, quella conoscenza si perde alla chiusura della conversazione.

Il Session Report risolve questo problema: dopo ogni sessione rilevante, viene creato un file che riepiloga cosa è stato fatto, cosa è stato deciso, cosa resta da fare.

---

## Quando creare un Session Report

Non ogni sessione richiede un report. Un report va creato quando in una sessione:
- È stato **prodotto codice** che è entrato in produzione
- È stata **presa una decisione strategica** (anche solo tra le due opzioni discusse)
- È stato **pubblicato un contenuto** (articolo, newsletter)
- È stato **risolto un problema critico**
- È stata **creata nuova infrastruttura** (sistemi, processi, strutture)

---

## Formato del Session Report

```
session-report/
├── 2026-06-26-newsletter-sistema.md
├── 2026-07-01-knowledge-core-v1.md
└── ...
```

Ogni file segue questa struttura:

```markdown
# Session Report — [Data] [Titolo breve]

**Data:** AAAA-MM-GG  
**Durata stimata:** X ore  
**Partecipanti:** CEO, AI (Claude)

## Cosa è stato fatto

Elenco delle attività completate nella sessione.

## Decisioni prese

Elenco delle decisioni rilevanti con motivazione.

## Problemi risolti

Cosa non funzionava, come è stato risolto.

## Cosa resta da fare

Task aperti emersi durante la sessione.

## File modificati / creati

Lista dei file principali toccati.
```

*Contenuto da popolare — Il primo Session Report verrà creato al termine di questa sessione.*
